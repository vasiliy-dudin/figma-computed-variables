# Project Context — Figma Computed Variables Plugin


## What This Project Is

A Figma plugin built with TypeScript that provides dynamic expression evaluation for design tokens.
The plugin has two runtime environments: the Figma sandbox (plugin code) and the UI iframe.
They communicate exclusively through `figma.ui.postMessage` / `parent.postMessage`.

## Problems Solved

### 1. Figma Variables Don't Support Computations

Native Figma Variables only support static values and simple aliases. There's no way to write `{spacing/base} * 2` or `alpha({color/primary}, 50%)`. Designers must manually calculate every value and update dozens of variables when a base token changes.

**Plugin solves:** JSON expressions with math, color functions, and string concatenation are automatically evaluated and written as values to Figma Variables.

### 2. No Single Text-Based Source of Truth

Figma Variables live only inside Figma. They can't be version-controlled in Git, can't be code-reviewed, can't be loaded from an external file. Tokens Studio solves some of these problems but locks you into its ecosystem.

**Plugin solves:** A JSON file is the single source of truth. It can be stored in Git, reviewed, generated by scripts. The plugin reads JSON and writes the result to Figma Variables.

### 3. Hard to Start from Scratch AND Hard to Start with Existing Variables

If a designer has no JSON file — they need to write one from scratch. If they already have Variables in Figma — there's no way to export them into an editable format.

**Plugin solves:** Import from Variables reads existing Figma Variables and converts them to the plugin's JSON format, including aliases. Getting started takes 30 seconds.

### 4. No Support for Three-Layer Token Architecture

Professional design systems build tokens in three layers: foundation (primitives) → semantic (meanings) → components (specifics). Native Figma Variables support collections but don't enforce this structure and don't simplify cross-layer dependencies.

**Plugin solves:** The JSON structure organizes tokens by collection (foundation / semantic / components), and expressions link layers together.


## Technology Stack

- **Language:** TypeScript (strict mode)
- **Plugin API:** Figma Plugin API
- **UI Framework:** Preact + `@create-figma-plugin/ui` components
- **Build Tool:** Vite
- **No backend. No database. No external API calls at runtime.**

## Project Structure

```
src/
├── plugin/          # Figma sandbox code (runs in Figma main thread)
│   ├── main.ts      # Entry point for plugin logic
│   └── ...
├── ui/              # UI iframe code (Preact components)
│   ├── app.tsx      # Main UI component
│   └── ...
├── shared/          # Types and utilities shared between plugin and UI
│   └── types.ts
└── expressions/     # Expression parser and resolver
    ├── parser.ts
    ├── resolver.ts
    └── ...
```

## Architecture Rules

- Plugin ↔ UI communication: ONLY through postMessage. No shared state, no global variables.
- All message types must be defined in `shared/types.ts`.
- Expression parser uses recursive descent. Do NOT replace with regex-based parsing.
- Keep plugin code and UI code strictly separated. No Figma API calls from UI code.

## What NOT to Touch Without Asking

- `manifest.json` — plugin configuration
- Build configuration files
- The expression parser core algorithm


## User Flows

### Flow 1: First Launch (Existing Variables in Figma)

```
Open plugin → click Import from Variables
→ plugin reads all Figma Variable Collections
→ converts to JSON with aliases {collection.token}
→ JSON appears in CodeMirror editor
→ user edits, adds expressions
→ clicks Apply → plugin writes back to Figma
```

### Flow 2: First Launch (Starting from Scratch)

```
Open plugin → empty editor
→ write JSON tokens manually or paste a template
→ click Apply → plugin creates Collections, Modes, and Variables
→ verify result in Figma → go back and refine
```

### Flow 3: Everyday Workflow

```
Open plugin → auto-loads last JSON from clientStorage
→ edit tokens (change values, add new ones)
→ click Apply → validation → if errors: banner + buttons disabled
→ fix errors → Apply → Figma Variables updated
```

### Flow 4: Updating a Base Token

```
Change {foundation.spacing/base} from 8 to 10
→ Apply → all dependent expressions ({spacing/base} * 2, * 1.5, ...)
→ automatically recalculated and written to Figma
→ one JSON edit updates dozens of variables
```
